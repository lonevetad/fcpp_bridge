KEY DESIGN DECISIONS
═══════════════════════════════════════════════════════════════════════════

Dynamic node dict:
  All examples now use dict[int, state] where node presence = dict key presence.
  Nodes can join/leave by adding/removing keys from the returned dict in round_step.
  AbstractExample.run() handles log file open/close when nodes appear/disappear.

Extra log files (receiver_messages.log, comm_receiver_messages.log):
  Opened in on_simulation_start(), written inside round_step() or via inline
  logic, closed in on_simulation_end().  AbstractExample.run() calls all hooks
  at the right points.

Cross-round state (in_flight, distances, etc.):
  Stored as instance attributes (self._...) initialized in __init__ or
  on_simulation_start().  This is cleaner than passing them through return tuples.
