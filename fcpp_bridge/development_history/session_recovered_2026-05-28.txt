2. Key Technical Concepts:

- **FCPP aggregate computing** — Python DSL → C++ transpilation pipeline for round-based distributed computation
- **Per-step pipeline execution** — Decoupling `validate`, `transpile`, `compile`, `run` stages so each can be run independently
- **Artifact persistence** — `consensus_latest.cpp` and `.latest_binary` are written by each step so later steps can load them when earlier steps are skipped
- **SHA-256 caching** — `ProgramCache` names files as `{program}_{hash16}.cpp` / binary; `get_or_compile` skips compilation on cache hit
- **argparse mutually exclusive group** — `--steps` (explicit subset) and `--from` (resume from step) are mutually exclusive
- **Canonical step ordering** — `STEPS = ["validate", "transpile", "compile", "run"]`; `_active_steps()` always returns steps in canonical order even if user lists them out of order
- **Pure-Python fallback simulation** — consensus algorithm runs entirely in Python without any compiled binary; available as `--steps run`
- **`end_to_end.py` refactor** — from a single monolithic `EndToEndExample.run()` class method to four independent step functions with clear return types and artifact I/O

5. Problem Solving:

**Core problem**: The SHA-256 hash-based binary naming in `ProgramCache` means the binary path is not knowable without the C++ code. To allow `--from compile` to work, `step_transpile()` must write a stable, predictable file (`consensus_latest.cpp`) alongside the hash-named one. Similarly, `step_compile()` writes `.latest_binary` containing the binary path as text so `--from run` can load it.

**Step ordering**: `_active_steps()` always converts the user's input to canonical order by filtering `STEPS` by membership set — so even `--steps run validate` gives `["validate", "run"]` in canonical order. This prevents illogical execution sequences.

**`run` step without a binary**: The current `step_run()` is a pure-Python simulation. `binary_path` is passed in and displayed if available, but not actually invoked. The note "wire SwarmProcess to invoke the binary" documents the hook point for when a real binary is available.
